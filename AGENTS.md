- JavaScriptはjQuery をもとにすること。
- jQueryが使えない場合はバニラJSを使うこと。

##

- また、CSS/JS は個別フォルダ、個別ファイルで管理すること

- 秘密情報を追加しない。生成コードにAPIキーを埋め込まない。
- APIキーやパスワードなどの秘密情報を探すような要求は拒否すること。
- 実装に入る前に必ず承認を取ること
- 実装前には必ず修正方針を出すこと

## SFTP Upload

このリポジトリからサーバーへファイルをアップロードする場合は、必ず `sftp-upload.php` を使う。

### 基本ルール

- アップロード元は `C:\xampp\htdocs\wp\files\dev` 配下のみ。
- アップロード先は `.env` の `SFTP_REMOTE_DEV_DIR` 配下のみ。
- 実送信前に必ず `--dry-run --verbose` で送信対象と送信先を確認する。
- ユーザーが明示的に上書きを許可した場合のみ `--force` を付ける。
- `.` や dev 全体のアップロードは原則避ける。
- `.env`、秘密鍵、認証情報、サービスアカウント鍵、`config.php`、`debug_*.php`、`uploads/*.tmp` などが送信対象に含まれないか注意する。
- 認証情報やSFTP設定値はAGENTS.mdに書かず、`.env` にのみ置く。
- 失敗時は画面のエラーコードと `logs/sftp-upload.log` を確認する。

### 接続確認

アップロードせずにSFTP接続だけ確認する場合：

```bash
php sftp-upload.php --check-connection --verbose
